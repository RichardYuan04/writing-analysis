②写手到底在比写作页的什么

从多篇「功能对比表」笔记里，反复出现的横向对比维度——这就是用户心智里写作页的「打分项」。

对比维度含义出现频率
小黑屋 / 强制写作模式设定字数或时间目标，未达成不可退出/删除，锁死专注几乎每篇对比表都列为独立一栏 ★★★
导出长图把正文一键排版成可晒的长图（模板/字体/颜色/自动九宫格切图）极高频 ★★★
多端同步 + 本地备份手机/平板/电脑/网页同步，自动本地备份防丢极高频 ★★★
是否「喂 AI」多款 App 把「官方承诺不喂 AI / 不拿你的稿训练」当卖点反复出现，强隐私信号 ★★
码字数据 / 速度今日码字、速度（字/分）、时长，并衍生游戏化高频 ★★
辅助工作区大纲/角色/设定/伏笔/时间线侧栏网文向高频 ★★
信号最强的两点：① 用户把「专注/防分心」当成写作页的核心功能（小黑屋几乎人人列）；② 大量 App 主动标注「不喂 AI」——说明写作者对「我的文字被拿去训练 AI」高度敏感。这直接利好你「用户自带 Key、隐私优先」的定位。

③写作页设计模式库（6 类）

把所读截图里的设计手法拆成 6 个可复用模式族。每条标注典型出处。

A · 专注 / 沉浸

小黑屋 / 强制写作

设定字数或时间目标前，禁用退出与删除，逼自己往前写。
出处：百灵创作、WonderPen、锦鲤写作、好好码字

可收起三栏

左目录 / 中编辑 / 右辅助，专注时一键隐藏两侧只剩正文。
出处：口袋写作、Obsidian、WonderPen

暗色 · 打字机居中

深色护眼主题、正文居中、极简无干扰（很多写手干脆用 VSCode 写小说）。
出处：VSCode 写小说、Obsidian 暗色主题

流逝模式

写完即「流逝」、降低「写得不好」的心理负担，鼓励自由表达。
出处：八度幻想「流逝模式」

B · 码字数据 & 游戏化

实时数据条

今日码字 / 本次码字 / 当前速度（字/分）/ 码字时长 / 空闲时长 / 摸鱼次数。
出处：百灵创作右侧数据面板

码字目标 + 进度

默认章节目标（如 3000）、今日目标字数（如 6000），状态栏实时进度条。
出处：Obsidian WebNovel Assistant、好好码字

码字房间 / 上墙台

限时冲刺房间：设目标字数、循环计时、可多人同屏一起冲。
出处：百灵创作「码字房间」

签到养成

每日签到「养一条锦鲤」，把坚持写做成养成游戏。
出处：锦鲤写作

日历码字热力

月历上每天码字量着色，连续天数一目了然。
出处：百灵创作数据总览（与你已有热力图同源）

C · 辅助工作区（网文向）

大纲 / 角色 / 设定 / 伏笔 / 时间线

左侧多 Tab 工作区，单独管理结构化创作资产。
出处：百灵创作、云上写作、Obsidian 插件

素材 / 灵感收集箱

随手存灵感碎片，再「编织」成情节。
出处：故事织机、橙瓜码字、幕布

关系图谱 / 思维导图 / 故事地图

把人物关系、世界观可视化。
出处：云上写作、幕布

敏感词检测 · 取名助手

发布前查敏感词、一键生成人名。
出处：锦鲤写作

D · 排版 & 导出（分享驱动）

一键排版 · 导出长图

正文套模板成图，可调字体/颜色/宽度，自动分割九宫格。
出处：番茄作家助手、Zine、稿定、纯纯写作（自动分九图）

选中文字改色 / 高亮

在编辑器里圈选局部文字改颜色、做标记。
出处：百灵创作、锦鲤写作

E · AI 辅助（需谨慎）

卡壳灵感生成

按题材/人设/风格一键生成：小说灵感 + 章节大纲 + 一键成文，还有「拆文/单元设计/原著仿写」。
出处：量子探险

编辑器侧接 AI

右侧常驻 AI 面板（直接接 Claude）做续写/分析。
出处：Obsidian WebNovel Assistant + Claude

⚠️ 反向信号：不喂 AI 是卖点

多款 App 强调「永久不喂 AI」。结论：AI 能力必须可选、透明、隐私优先，不能默认上传内容。
出处：好好码字、口袋写作

F · 习惯 / 仪式 / 内省 和你最契合

提问引导内心写作

不为发表，用提问把你引向内心；「仅你可见，诚实去写」；培养写作习惯。
出处：Koan

动态欢迎语

创作主页按时段变化的欢迎语 + 启示寄语 + 今日新增数据。
出处：Obsidian WebNovel Assistant 创作主页

梦境 / 日常想象记录

把灵感当日记记，每条配图，弱目标、低压力。
出处：八度幻想

「如何进入状态」是刚需

「写文怎么进入状态」类内容高赞——写作者把「进入心流的仪式」当真实痛点。
出处：多篇高赞经验帖

⑤映射到「文字时光机」的建议

你的产品是个人随笔 + AI 分析，不是网文工厂。所以不能照搬，要按契合度过滤。

✅ 高契合 · 值得直接借鉴 FIT
  • 专注/沉浸模式：做一个轻量「专注写作」——隐藏导航、暗色/护眼、正文居中打字机。（小黑屋的「强锁」对随笔太硬，取其「沉浸」舍其「强制」。）
  • 码字实时数据 + 游戏化：你已有热力图，天然可延伸出「今日字数/连续天数/本次时长」。可把网文的「摸鱼次数/码字房间」温和化成「专注时长/今日心绪打卡」。
  • 动态欢迎语 + 进入状态仪式：写作页打开时按时段给一句话 + 今日提示，降低空白页焦虑——正好补你现在「纯 textarea 无引导」的短板。
  • 提问引导（Koan 式）：空白页时给一个「今天想记录什么」的轻提问/选题，呼应你的「写作画像」。
  • 自动保存 + 本地备份：这是写作页红线，几乎所有 App 都做了，你现在缺——应作为基础体验优先补。
  • 导出长图分享：把你独有的「分析/情感卡片」做成可晒长图，呼应你已有的 ljg-card 能力。
⚠️ 谨慎 · 不照搬 CARE
  • 网文工作区（大纲/角色/设定/伏笔/敏感词/取名）：对个人随笔过重，会把产品拖向网文工具。除非你想拓展到「长文/小说」人群，否则跳过。
  • 强制小黑屋：随笔写作不该有「写不够不让走」的压迫感；做成可选的「专注模式」即可。
  • 重排版导出引擎：番茄/Zine 那种复杂排版模板系统投入大，先用「一张漂亮的心绪卡」覆盖 80% 分享需求。
🌟 差异化机会 · 别人没有的牌 EDGE
  • 写完即分析的闭环：所有竞品的写作页都是「哑输入框」，分析都在别处。你独有「画像 + 情感时间线 + 深度分析」——把它前置到写作页，保存后立刻给一张「今日心绪卡 + 和过去的对照」，是没人做过的体验。
  • 把「码字数据」升级成「心绪数据」：竞品统计字数/速度，你可以统计情绪走向、关键词、与历史的呼应——同样的仪表盘骨架，灌入你独有的分析。
  • 隐私即定位：写手群体对「不喂 AI」高度敏感，而你正是「用户自带 Key、本地优先」。把这点显性化（写作页标注「你的文字只走你自己的 Key」）会直接戳中人群。

⑥下一步

这份研究是 PM 链路的「外部输入」。建议接回已部署的 PM skill 流程：

  • problem-statement → 用本报告的洞察，把写作页要解决的问题框准（如「空白页焦虑 / 哑输入框 / 写作-分析断裂」）
  • jobs-to-be-done → 区分「网文写手 job」与「你的随笔用户 job」，避免被竞品带偏
  • opportunity-solution-tree → 把第 ⑤ 节的 FIT / EDGE 长成机会树
  • prioritization-advisor → 在「补红线（自动保存）/ 专注模式 / 写完即分析闭环」里排序
  • prd-development → 出可落地 PRD（配设计稿，符合「设计稿优先」）

我的快速判断：先补「自动保存 + 本地备份」红线（低风险高价值），同时把「写完即分析闭环」作为这一轮的差异化亮点功能——这两件最值得先做。
