# Design: 文字时光机 · 国内发布部署与商业化设计

**日期**: 2026-06-25
**状态**: 待评审
**范围**: 把现有「单机自用」的 FastAPI + React app 改造成面向**国内消费者**、可承接多用户、可收费的 SaaS。覆盖：多租户数据改造、账号/支付/计量封顶、全国产模型迁移、国内云部署拓扑、内容合规、分阶段落地。

> 本 spec 只定**方向与架构**，不含逐行实现。通过评审后再拆实现计划（writing-plans）。

---

## 1. 背景与目标

### 现状
- 后端单文件 FastAPI + SQLite（`essays.db`），所有数据表**无 `user_id`**，SOUL 为写死单行 `id=1`，即**单租户**。
- AI 调用混用 Claude（Haiku/Sonnet/Opus）+ DeepSeek + Gemini + 两个**本地 ML 模型**（情绪分类、embedding 向量）。
- 后端无认证、无计量、无支付；前端 React/Vite 纯静态。

### 目标
1. **多租户**：每个用户的随笔/草稿/SOUL/半成品彼此隔离。
2. **可发布**：部署到国内云、走完备案与内容合规、用户能注册登录。
3. **可收费**：免费档（限量）+ 订阅解锁，按用户计量、配额封顶以控成本。
4. **全国产模型**：去除跨境出境调用（成本、稳定性、合规三重收益）。

### 非目标（本期不做）
- 团队/协作、多端 App（仅 Web）。
- 海外版（出境网络、USD 计费、Claude 回归——列为后续）。
- 复杂的用量计费（按 token 精算账单）——本期用「调用次数配额」这种粗粒度即可。
- 半成品仓库 / SOUL 等功能本身的玩法迭代（属于产品线，不在部署 spec 内）。

## 2. 整体架构（部署拓扑）

```
                         用户浏览器
                              │
                   ┌──────────┴───────────┐
                   ▼                      ▼
          静态前端 (React build)     API 后端 (FastAPI)
          阿里云 OSS + CDN          阿里云 ECS（常驻，含本地 ML 模型）
          （已备案域名）                  │
                                         ├─ RDS PostgreSQL（多租户主库）
                                         ├─ Redis（会话 / 限流 / 配额计数）
                                         ├─ 短信服务（注册验证码）
                                         ├─ 内容安全 API（UGC + AI 输出审核）
                                         ├─ 微信支付 / 支付宝（订阅）
                                         └─ 国产模型 API（DeepSeek 为主）
```

- **前端**：纯静态，OSS + CDN 托管，几乎零运维。
- **后端**：**一台常驻 ECS**（≥2GB 内存，因本地 ML 模型进程内加载）。MVP 单机；扩容路径见 §11。
- **数据**：SQLite → **RDS PostgreSQL**；**Redis** 承担会话、限流、配额计数（高频读写不压主库）。
- 云厂商：**阿里云**为基准（腾讯云等价可替）。

---

## 3. 多租户数据改造（最大工程量，先行）

这是从「单机」到「SaaS」的根，必须最先做。

### 3.1 新增 `users` 表
`id, phone, email, wechat_openid, nickname, created_at, plan('free'|'pro'), plan_expire_at`（三种登录标识允许为空，至少一项非空）。

### 3.2 所有业务表加 `user_id` + 外键 + 索引
- `essays`、`drafts`、`fragments`、`theme_clusters` 全部加 `user_id`，所有查询加 `WHERE user_id = :uid`。
- **`style_profile` 从「单行 id=1」改为「每用户一行」**：主键改 `user_id`（或加唯一约束），`_load_soul_bundle()` / generate / get / put 全部按 `user_id` 取。
- `reader_letters`（现挂在 drafts/essays JSON 字段上）随宿主行天然隔离，无需单独改。

### 3.3 鉴权注入
- FastAPI 依赖项 `current_user`（从 session/JWT 解析），所有业务端点强制带上，**禁止跨用户访问**（越权防护）。
- 老的单机数据：提供一次性脚本把现有 `essays.db` 归到「站长账号」或丢弃（本地开发数据，倾向丢弃）。

> **风险标注**：这一步触及几乎每个端点和每条查询，是本项目最大的一块改造，必须配测试（每个端点的「越权访问返回 403/404」用例）。

## 4. 模型层：全国产迁移

### 4.1 本地 ML 模型 → **保留在后端进程内**
- 情绪分类、embedding 向量继续本地跑（零出境、零 API 费），随 ECS 常驻内存。
- 影响：ECS 需 ≥2GB 内存、首次启动加载数秒（用启动预热 + 健康检查覆盖）。

### 4.2 Claude/Gemini 调用 → 国产模型映射
逐功能替换（主力 **DeepSeek**，按需用通义/智谱补特性）：

| 功能 | 现模型 | 替换为 | 备注 |
|---|---|---|---|
| 缩减 | Haiku | DeepSeek | |
| 同义 / 扩展 | Sonnet | DeepSeek | |
| 比喻 | **Opus 4.8** | DeepSeek | 当前最贵点，降档收益最大；prompt 需重调 |
| SOUL 养成 | Sonnet | DeepSeek | 低频、吃质量，可考虑用更强的国产档 |
| 读者风格卡 | Sonnet | DeepSeek | |
| 读者视角 | DeepSeek | （不变） | |
| 半成品分类 / 主题命名 | Haiku | DeepSeek | |
| 心情卡等辅助 | Haiku/Gemini | DeepSeek | |
| **找引文** | Sonnet + **联网搜索** | 通义/智谱（带联网）**或** 博查/Bing 搜索 API + DeepSeek 组合 | **特别注意**：DeepSeek 无原生联网，此功能需单独方案 |

- 统一一个 `llm_call(purpose, ...)` 适配层，按 `purpose` 路由到模型 + 模型专属 prompt，便于将来再调档/再换厂商。
- **SOUL / 禁止项 / 黄金样例**这套 system 前缀是每次复用的固定内容 → 开启**模型侧 prompt 缓存**（DeepSeek 支持 context caching），砍输入成本。

### 4.3 成本预期
全国产后单用户月成本压到**几毛人民币**量级（典型用户）。详见前期讨论；订阅 ¥18–30/月毛利充足，重度用户靠 §6 配额封顶兜底。

## 5. 账号体系（三方式登录）

### 5.1 支持的登录方式
- **手机号 + 短信验证码**（主力，首发）：接阿里云短信；注册即实名基础。
- **邮箱 + 密码**（首发）：实现最简，作为备选入口。
- **微信登录（OAuth）**（快速跟进）：转化最丝滑，但需微信开放平台**资质审核**，作为首发后第一个跟进项，避免阻塞发布。

> 一个 `users` 行可绑定多种标识；同号/同邮箱/同 openid 视为同一账号（账号合并策略：以手机号为主标识）。

### 5.2 会话
- 登录后签发 **JWT**（或 Redis session），前端存储，后续请求带 `Authorization`。
- 过期 + 刷新机制；登出即失效。

## 6. 计量与配额封顶（控成本核心）

### 6.1 计量
- 每次 AI 调用记一条 `usage`（`user_id, purpose, model, tokens_in/out, cost_est, ts`）——**异步写**（先进 Redis 计数，批量落库），不拖慢请求。
- 维度：按用户、按功能、按天/月聚合。

### 6.2 配额（按「调用次数」粗粒度，免 token 精算）
- **免费档**：每月固定额度（如 AI 操作 N 次/月 + 后台分析随写作量但设上限）；用尽 → 提示升级、降级或排队。
- **订阅档（Pro）**：更高额度 + 解锁高成本功能（比喻、找引文、SOUL 养成等）。
- **硬封顶**：即便 Pro 也有月度上限，防滥用/脚本刷爆账单；触顶后当月限流。

### 6.3 限流中间件
- Redis 令牌桶：按 `user_id` 限 RPS / 并发，防单用户突发。
- 厂商侧速率上限（DeepSeek org RPM/TPM）：加**队列**平滑突发 + 申请提额；真正瓶颈在此，不在你的服务器。

---

## 7. 付费：免费档 + 订阅

- **支付渠道**：微信支付 + 支付宝（国内双标配）。
- **订阅模型**：月/年订阅，写 `users.plan` + `plan_expire_at`；到期降级为 free。
- **回调**：支付成功回调 → 验签 → 升级账号 → 幂等处理（防重复回调）。
- **对账**：`orders` 表记录每笔交易，供售后与财务核对。
- 退款/客诉流程本期最小化（人工处理）。

---

## 8. 内容合规（国内发布强制项）

国内公开 UGC + AI 生成内容**必须过审**，不可省：

- **用户输入**（随笔正文等）+ **AI 输出**双向接**内容安全审核**（阿里云内容安全 / 数美 / 绿盟等）：机审命中违规 → 拦截/打标/人审。
- **实名**：手机号注册已满足基础实名要求。
- **备案**：公开域名走 **ICP 备案**（+ 若涉及经营性可能需经营性 ICP/EDI，视收费形态确认）。
- **AI 生成内容标识**：按《生成式 AI 服务管理办法》要求，AI 产出需可标识/可追溯（日志留存 + 必要时显式标注）。
- **日志留存**：访问与生成日志留存满足监管要求的时长。

> 合规不是「锦上添花」，是发布前置闸门。低估它会卡上线。

---

## 9. 数据迁移（SQLite → PostgreSQL）

- SQLAlchemy 模型基本不变，改连接串 + 跑建表/迁移。
- 引入正式迁移工具（Alembic）替代现有 `migrate_db()` 手写 ALTER，多人多环境更可靠。
- 注意 SQLite 与 PG 的差异：布尔/整数、JSON 字段（PG 用 `jsonb` 更佳）、自增主键、并发事务。
- 现有本地 `essays.db` 仅开发数据，**不迁移**（或归站长账号），避免污染。

---

## 10. 可观测与运维

- **错误监控**：Sentry（或云监控）捕获后端异常。
- **成本看板**：基于 `usage` 表出「日/月 API 成本、按功能拆分、Top 重度用户」——盯住毛利。
- **健康检查 + 自动重启**：systemd / Docker restart；ECS 健康探针。
- **备份**：RDS 自动备份 + 定期快照。
- **密钥管理**：模型 key、支付密钥放云端密钥管理/环境变量，不入库不进前端。

## 11. 分阶段落地（里程碑）

**M1 — 多租户地基（最大块、先做）**
`users` 表 + 全表加 `user_id` + `current_user` 鉴权注入 + 越权防护测试 + SQLite→PG。无此则后面都无意义。

**M2 — 账号 + 会话**
手机号验证码 + 邮箱登录、JWT、注册登录流程。

**M3 — 全国产模型迁移**
`llm_call` 适配层、逐功能替换 + prompt 重调、找引文的联网方案、prompt 缓存。

**M4 — 计量 + 配额 + 限流**
usage 记录、免费/Pro 配额、Redis 限流、硬封顶。

**M5 — 支付 + 订阅**
微信/支付宝接入、回调验签、订阅状态、orders 对账。

**M6 — 合规 + 部署上线**
内容安全接入、ICP 备案、AI 标识、日志留存、ECS+OSS+CDN 正式部署、压测。

**M7（跟进）** 微信登录、成本看板打磨、扩容（拆本地模型 / 多实例 / Cloud Run 形态）。

---

## 12. 风险与开放问题

- **多租户改造面广**：触及几乎每个端点，越权是最大安全风险 → 强制每端点鉴权 + 测试覆盖。
- **找引文联网**：DeepSeek 无原生搜索，需单独选型（通义/智谱带联网 or 搜索 API + 模型），可能延后或降级。
- **合规时长不可控**：备案 + 内容安全接入 + 可能的经营性资质，是上线时间的主要不确定项。
- **国产模型质量回退**：比喻/SOUL 从 Opus/Sonnet 降到 DeepSeek，需重调 prompt 并人工验收质量，必要时这两项用更强国产档。
- **开放问题**：①是否需要经营性 ICP（取决于收费形态合规判定）；②免费档具体额度数字（需结合成本看板上线后调）；③账号合并策略细节（同人多登录方式如何归并）。

---

## 13. 已确认决策
- 受众：**国内**。
- 模型：**全国产**（DeepSeek 为主），本地 ML 模型**留进程内**。
- 登录：**手机号 + 邮箱首发，微信跟进**。
- 付费：**免费档（限量）+ 订阅解锁**，按调用次数配额 + 硬封顶控成本。
- 云：阿里云为基准；前端 OSS+CDN、后端常驻 ECS、数据 RDS PG + Redis。
