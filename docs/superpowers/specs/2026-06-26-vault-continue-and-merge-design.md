# Design: 半成品仓库 · 续写智能化（#1）+ 多片段/整主题合并续写（#2）

**日期**: 2026-06-26
**状态**: 待评审
**范围**: 把半成品仓库从「只读陈列」变成「创作起点」。两件事：
- **#1 续写智能化**：续写时把片段的 `ai_hint` 带进写作页当提示，并提供一键「让 AI 按你的风格接着写」。
- **#2 合并续写**：支持勾选多条片段、或「拿整个主题」，一起开一篇草稿。

> 本 spec 偏前端 + 一个新 assist 端点；**无数据库 schema 改动**。下方「开放决策」是请你重点反馈的地方。

---

## 1. 背景与目标

### 现状（已读代码确认）
- 卡片「续写」= `onContinue(fragment)` → `onWrite(fragment.content)` → `navigateToWrite(prefill)`，写作页把 `prefill`（**纯字符串**）经 `plainTextToBlocks` 铺成段落。仅此而已。
- `ai_hint`（续写建议）养成时算好、卡片上显示，但**没进编辑器、不驱动任何动作**。
- 主题视图（`ThemeClusterCard`）**只读**：点其中一条只能**单条**续写。
- 写作工具（扩展等）走 `_assist_call`、已注入 SOUL/禁止项/黄金样例，但续写那一刻没用上。

### 目标
1. 续写时携带 `ai_hint`，并能一键调 AI 生成贴合风格（走 SOUL）的接续草稿。
2. 多选片段 / 整主题 → 合并成一篇草稿的起点。

### 非目标
- **#3「已发展」闭环**（片段消化后打标）——本期不做，列为后续。
- 不改提炼/聚类质量（#5/#6 另议）。
- 不做协作/版本。

---

## 2. 统一的管线改动（#1 与 #2 共用）

把写作页入参 `prefill` 从**纯字符串**升级为对象：

```
prefill = { text: string, hints: string[] }
```

- **#1 单条**：`{ text: fragment.content, hints: fragment.ai_hint ? [fragment.ai_hint] : [] }`
- **#2 多条/整主题**：`{ text: 选中片段 content 以空行拼接, hints: 选中片段的 ai_hint 去空后数组 }`

兼容：`Write.jsx` 同时接受**字符串**（老调用、本地草稿恢复路径）与**对象**；字符串等价于 `{text, hints: []}`。`plainTextToBlocks` 已按 `\n` 切段，故 `text` 用 `\n\n` 拼接即天然分段。

**改动点**：`App.jsx`（透传对象）、`Write.jsx`（解析对象 + 渲染 hints + 接续按钮）、`DraftVault.jsx`（构造对象）。

## 3. #1 续写智能化

### 3.1 携带 `ai_hint`（提示横幅，不算正文）
写作页顶部、编辑器上方显示一条**可关闭的提示横幅**（仅当 `prefill.hints` 非空）：
> ✦ 来自半成品的提示：<hint1>；<hint2>…

它**不写进正文**，只作引导，用户可一键关掉。

### 3.2 一键「让 AI 接着写」（走 SOUL + hint）
横幅里带一个按钮 **「✦ 让 AI 接着写（按你的风格）」**：
- 取当前编辑器纯文本 + `hints`，调新端点 `/assist/continue`。
- 返回一段**承接原文、用作者声音**的续写，**插入到正文末尾**（作为新段落），用户可改/可撤。
- 加载态「正在接写…」；失败提示可重试。

**新后端端点** `POST /assist/continue`：
- 入参 `{ text, hints: string[], context?: string }`。
- 复用 `_assist_call` 基础设施 → 自动注入 SOUL 正文 + 禁止项 + 黄金样例。
- 续写专属 system：「你是该作者本人，接着已有文字往下写 2–4 句，承接语气与思路；参考提示但不照搬；不要重复已有内容、不要解释。」`hints` 作为软引导拼进 user。
- 模型：与扩展同档（当前 Sonnet→国产迁移后随 `_assist_call` 统一路由）。

## 4. #2 多片段 / 整主题合并续写

### 4.1 选择交互
- **类别视图**：进入「选择模式」（一个「多选」开关，或卡片长按/勾选框）；被选片段高亮。底部浮出操作条：**「已选 N 段 · 合并续写」**＋「清空」。
- **主题视图**：每个 `ThemeClusterCard` 头部加按钮 **「用整组开一篇」**（取该主题下全部**可见**片段）。

### 4.2 合并铺入编辑器
- 选中片段的 `content` 按**点选顺序**以**空行**拼接为 `prefill.text`；其 `ai_hint` 收进 `prefill.hints`。
- 走 §2 同一条管线进写作页；#1 的提示横幅与「让 AI 接着写」对合并结果同样适用（自然组合）。

### 4.3 约束
- 软上限 **≤8 段**（超出提示「选得有点多，建议精简」），防超长 prefill 与跑题。
- 「整主题开一篇」**仅含可见片段**（隐藏的不带入）。

## 5. 前后端改动清单

**后端**（`backend/main.py`）
- 新增 `POST /assist/continue`：续写专属 system + `_assist_call`（注入 SOUL/禁止项/黄金样例）。
- `backend/tests/`：续写端点注入断言（system 含 SOUL/禁止项、含 hint）。

**前端**
- `api.js`：`assistContinue({text, hints, context})`。
- `App.jsx`：`onWrite`/`navigateToWrite` 透传 `{text, hints}` 对象。
- `Write.jsx`：解析对象型 `prefill`；提示横幅 + 「让 AI 接着写」按钮 + 末尾插入逻辑。
- `DraftVault.jsx`：类别视图多选态 + 浮动操作条；`ThemeClusterCard` 「用整组开一篇」；构造 `{text, hints}`。
- `App.css`：选择态高亮、操作条、提示横幅样式（沿用主题变量，避免内联堆叠）。

## 6. 开放决策（请你重点反馈）

1. **AI 接着写：一键触发 vs 自动跑**——推荐**一键**（省 API、可控，避免每次续写都花钱）。
2. **续写用新 `/assist/continue` vs 复用 expand**——推荐**新端点**（语义不同：续写=往下生成，扩展=就地铺陈一段选区）。
3. **`ai_hint` 呈现：提示横幅 vs 注释块插进正文**——推荐**横幅**（不污染正文，可关闭）。
4. **合并顺序与分隔**——推荐**按点选顺序 + 空行**；备选按质量分或日期排序。
5. **多选上限**——推荐软上限 **8 段**；可调或不限。
6. **AI 接着写的结果如何落地**——推荐**直接插入末尾可编辑**；备选「先展示、点采用再插入」（更稳但多一步）。
7. **选择交互形态**——勾选框 / 卡片点击切换 / 「多选」开关，哪种你更想要（影响卡片现有「续写·隐藏」按钮的布局）。

## 7. 测试
- 后端：`/assist/continue` 注入断言（system 含 SOUL 正文片段、禁止项关键词、hint 文本）；无 SOUL 时降级仍带 DEFAULT_TABOO。
- 前端：手动走查——单条续写见横幅+接写；多选 3 段合并成一篇；整主题开一篇；上限提示。
