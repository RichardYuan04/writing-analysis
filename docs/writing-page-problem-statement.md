# 写作页迭代 · 问题陈述（Problem Statement）

> 框架：Jobs-to-be-Done + 共情地图（problem-statement skill）
> 状态：**待校验草稿** — 基于应用现状 + 小红书竞品研究 + 产品主理人视角；尚无正式用户访谈，`Which makes me feel` 一节需你确认是否真实。

## Problem Framing Narrative

**I am（我是）：** 一个习惯用写字来梳理自己、偶尔也想把它打磨成作品的人
- 主场景写日记 / 随笔 / 碎想，**不为发表，为了记录和理解自己**
- 但也有**轻创作**需求：偶尔把随笔扩成短文 / 公众号草稿 / 想认真写的一篇（双重身份）
- Mac 用户，多半试过备忘录 / flomo / Notion / 纸本，但大多半途而废
- 在意隐私——**不想自己的私人文字被拿去训练 AI**

> **persona 已确认（2026-06-07）**：随笔 + 轻创作双场景。随笔是地基、轻创作是延伸；不做重网文工作区，但「从随笔顺手长成一篇」的路径要留。

**Trying to（我想做到）：**
- 持续地把日常的想法和情绪写下来，并且**能回头看见自己的变化**——理解「我是谁、我正在经历什么」

**But（但是）：**
- 打开写作页是个空白框，**不知从何写起**（空白页焦虑）
- 写的时候容易分心、被打断，**进不去状态**
- 写完保存就跳走了，**看不到任何即时回馈**——写这件事缺乏「值得」的感觉
- 写一半的内容**容易丢**（无自动保存 / 无草稿保护）
- 写完的文字和应用其余的智能（画像 / 情感时间线 / 热力图 / 深度分析）是**割裂的**——落笔的当下感受不到分析的价值

**Because（根因）：**
- 写作页被当成一个纯粹的「输入端口」在做，而不是一个「陪你进入状态、并在落笔后立刻回馈意义」的场所
- 应用把所有智能都放在了**写完之后的分析页**，于是**写作的当下是空的**

**Which makes me feel（我的感受）：**
- 对着空白框提不起劲、有点孤单
- 写了也不太清楚「为什么写」
- 于是慢慢就不写了 —— **静默流失**

---

## Context & Constraints（背景与约束）

- **场景**：个人随笔为主 + 轻创作延伸（明确区别于小红书主流的重度网文写手人群）
- **隐私**：用户自带 API Key、本地优先（桌面 App 方向）
- **平台**：Mac 优先、桌面端
- **工程约束**：迭代期停在 Phase 0（FastAPI 同源单进程托管），**不宜大重构**
- **协作习惯**：UI 改动**设计稿优先**，批准后再写代码

---

## Final Problem Statement（最终问题陈述）

> 习惯用写作梳理自己的人，需要一个能**陪他进入状态、并在落笔后立刻回馈「这段文字意味着什么」**的写作页；而现在的写作页只是个**会丢内容、写完即走的空白输入框**，让人渐渐失去持续写下去的动力。

---

## 三个候选「主问题」（需排序，进入 jobs-to-be-done 前先聚焦）

1. **捕捉摩擦**：空白框 + 无引导 + 会丢内容 → 写不顺、不敢写、留不住
2. **写作-分析断裂**：落笔当下感受不到应用最值钱的智能 → 写作缺乏即时意义回馈
3. **持续动力缺失**：没有专注/仪式/反馈循环 → 写几次就停了，静默流失

> **已锁定（2026-06-07）：这轮主问题 = 1 + 2 一起做。**
> - 问题 1（捕捉红线）：自动保存 / 防丢 / 选题引导 —— 低风险必补的地基
> - 问题 2（写作-分析闭环）：落笔即回馈意义 —— 这轮的差异化亮点
> - 问题 3（持续动力）：由 1+2 自然缓解，不单独立项
> 下一步用 jobs-to-be-done 把这两条拆成可验证的用户 job。
